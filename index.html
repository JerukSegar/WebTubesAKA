Merge Sort Rekursif (Top-Down)

Paradigma: Divide and Conquer

Kompleksitas Waktu: O(n log n)

Kompleksitas Ruang: O(n) + O(log n) untuk stack

Implementasi: Fungsi rekursif membagi array hingga elemen tunggal

Overhead: Panggilan fungsi rekursif dan manajemen stack

Keterbatasan: Risiko stack overflow untuk data sangat besar

O(n log n) - Stable Sort

Merge Sort Iteratif (Bottom-Up)

Paradigma: Bottom-Up

Kompleksitas Waktu: O(n log n)

Kompleksitas Ruang: O(n)

Implementasi: Loop iteratif menggabungkan sub-array bertahap

Overhead: Manajemen indeks dan variabel loop

Keunggulan: Tidak ada batasan stack, lebih hemat memori

O(n log n) - Stable Sort
